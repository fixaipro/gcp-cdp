STEP 01
Signal Collection
GA4 web + Firebase app events fire with Customer ID. Offline CSV ingestion runs nightly. MMP sends S2S attribution events. All data lands in GCP Cloud Storage staging bucket.
STEP 02
Identity Resolution
Identity Graph runs deterministic matching on Customer ID. Probabilistic matching fills gaps. 2P data reconciled. Single customer view written to BigQuery master table.
STEP 03
Consent Filtering
CMP consent signals applied. Only consented profiles proceed downstream. Suppression lists maintained. GDPR DSR processing within 72hrs target SLA.
STEP 04
Model Execution
BigQuery ML runs CLV, propensity, churn and NBA models on enriched profiles. Outputs scored back to BigQuery as model result tables. Triggered nightly via Cloud Scheduler → Dataflow.
STEP 05
Audience Activation
Segmented audiences pushed via Connector Hub to ad platforms. Hashed emails/phone for Customer Match. Match rates and scale surfaced in Audience UI. Target: 7M+ segment capacity.
STEP 06
Measurement & BI
Looker reads from BigQuery views. Shapley attribution model joins paid media spend to conversions via Customer ID. Econometrics model receives enriched base data. Reports auto-refresh daily.

Security & Privacy Architecture

Privacy-First by Design

Every data product built on a foundation of privacy compliance, consent-first data collection, and enterprise-grade GCP security controls.

🔐
Consent Management

All data collection gated through a Consent Management Platform (CMP). Only explicitly consented signals flow downstream to BigQuery, identity resolution, and ad platform activation.

  • GDPR Article 7 compliant consent capture
  • PECR ePrivacy regulation adherence
  • Consent signals propagate to all downstream DPs
  • Suppression lists for opted-out IDs
  • DSR processing target <72hrs

🏗️
GCP Infrastructure Security

All data within GCP's SOC 2 / ISO 27001 certified environment. Service accounts, IAM roles and VPC Service Controls enforce least-privilege access at every layer.

  • BigQuery column-level access controls
  • Service account-based authentication (no user credentials)
  • VPC Service Controls — data perimeter enforcement
  • Cloud KMS encryption at rest and in transit
  • Cloud Audit Logs for all data access events

🪪
Identity & Data Minimisation

No PII shared with ad platforms in raw form. All identity resolution uses hashed signals. Probabilistic matching used for aggregated insights only — never individual targeting.

  • SHA-256 hashed email + phone for Customer Match
  • Pseudonymised Customer ID (not raw PII)
  • No device IDs or third-party cookies at any layer
  • 2P data via clean room (no raw data share)
  • Data minimisation per DP Privacy Impact Assessment (PIA)

📋
Governance & Compliance

Each data product has an associated PIA and SIA before production deployment. Change management tracked centrally. Data lineage documented in BigQuery Data Catalog.

  • PIA / SIA required for every DP before go-live
  • Data lineage tracked via BigQuery Data Catalog
  • Retention policies enforced via GCP lifecycle rules
  • Third-party data processor agreements in place
  • Annual privacy audit scheduled

🔄
Server-Side Data Flow

Server-side data collection (CAPI / Event Streams) replaces client-side pixel tracking, eliminating browser-based interception risk and ensuring data integrity source to BigQuery.

  • S2S CAPI for Meta & Google signals
  • HTTPS-only endpoints for all data ingestion
  • Server-side tag management (Event Streams)
  • No third-party JavaScript on conversion pages
  • Signed requests for all Connector Hub API calls

🧪
Testing & Validation

Every data product goes through pre-production testing with data quality checks, schema validation, and Test & Learn framework design before activation budget is committed.

  • Pre-prod data quality scoring before promotion
  • Schema validation enforced by Dataform / dbt
  • A/B holdout groups for all audience segment tests
  • Incrementality measurement for all new DPs
  • Analytics team Test & Learn sign-off required

Zero Cookie Dependency: The entire architecture operates without third-party cookies, device IDs, or browser fingerprinting. First-party Customer ID is the single thread connecting all data products — making the platform durable against any future browser or OS privacy changes.

Looker Architecture

Looker as Single Source of Truth

Looker connects natively to BigQuery — no data copies, no ETL lag. LookML defines all metric definitions centrally.

📐 LookML Model Structure

+

Three core Explores: Customer Journey (touchpoint-level), Campaign Performance (channel/campaign level), Product Conversion (product × channel × segment). All metrics defined once in LookML.

explore: customer_journey {
  join: attribution_model { type: left_outer }
  join: channel_spend { type: left_outer }
  join: product_conversion { type: left_outer }
  join: clv_scores { type: left_outer }
}

🔗 BigQuery → Looker Data Flow

+

BigQuery materialised views refresh nightly via scheduled queries. Looker connects via read-only service account. PDTs (Persistent Derived Tables) pre-aggregate heavy attribution calculations so dashboards load in <3s. Looker Scheduler distributes daily performance reports automatically to channel teams.

📊 Key Attribution Dashboards in Looker

+
  • Cross-Channel Attribution — Shapley credit by channel, campaign, creative. Filters by product, segment, date.
  • Customer Journey Explorer — Touch path visualisation showing sequence of channels before conversion.
  • Budget Optimiser View — Marginal ROAS by channel. Surfaces over/under-invested channels.
  • Audience Performance — CLV segment performance across channels. LTV × channel interaction.
  • Econometrics Input Feed — Standardised output for Marketing Mix Model. Daily spend, impressions, conversions.

⚡ How Shapley Attribution Works in BigQuery

+

Shapley value attribution uses cooperative game theory to distribute conversion credit across all channels in a customer's journey. Each channel's marginal contribution is measured — how much worse the conversion rate would be without that touchpoint.

In BigQuery, this runs as a stored procedure evaluating all 2ⁿ channel subsets and computing average marginal contribution per channel. Results written to attribution_results table and exposed in Looker.

WHY SHAPLEY > LAST-CLICK

In telecoms, average customer journeys span 8–12 touchpoints over 30–60 days before contract conversion. Last-click over-credits brand search and under-credits prospecting (TV, social, display). Shapley corrects this — typically shifting 15–25% of attributed budget from lower-funnel to upper-funnel when first deployed.
